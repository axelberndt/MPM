1.8. <dynamicsMap>

<dynamicsMap> Allowed in global or local context as child of dated-element. Contains dated dynamics instructions. Preprocessing has to make sure a global dynamicsmap exists for every style, and pass them to lokal contexts (part-element) if no dynamicsmap is present there. 1.23. <viennaCtrls/>

<viennaCtrls/> Vienna Instruments allow control of "Velocity-Cross-Fade", that influences velocity, expression, ChannelVolume and tone color accordingly. It can be addressed via specific controllers. To provide the engine with the right controller data this element allows the corresponding encoding. On performance processing a global standard-tag will be generated if none exists in order to provide one for each part, with @velXFade=20 and @velXFadeOnOff=20.

3.2. att.dynamics

att.dynamics Group of dynamics attributes.
Moduledynamics
Membersdynamics
Attributes
curvatureOptional. Indicates the deviation from a linear volume transition/envelope. Potential values: [0] linear envelope; [1] steep envelop on the curves turning point; [0.3] default value.
StatusOptional
Datatype
volumeGives the volume either as defined in parent::dynamicsmap attributes or as absolute MIDI velocity (0 to 127).
StatusOptional
Datatype
transitionGives the target dynamics/volume either as defined in parent::dynamicsmap attributes or as absolute MIDI velocity (0 to 127).
StatusOptional
Datatype
protractionOptional. Indicates how long the level adjustment will be delayed. Value rang -1 to 1. Values smaller than [0] provoke a quicker change at the beginning, values larger than [1] provoke a quicker change at the end of the transgression.
StatusOptional
Datatype
subNoteDynamicsGives the density of ChannelVolumeChange-commands. This resembles the slope of 't' in calculating the bezier curve, e.g. for(t = 0; t lte 1; t = t + subNoteDynamics) {calculate X-Y-coordinates at t;}. If no subnote dynamics are wanted, the value is set to 0, 1, or left away
StatusOptional
Datatype
